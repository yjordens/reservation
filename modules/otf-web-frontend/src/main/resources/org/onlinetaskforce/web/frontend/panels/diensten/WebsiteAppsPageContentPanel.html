Mobile apps ontwikkeling

Je kan er niet omheen. De smartphone en tablets nemen alsmaar meer de plaats in van desktops en laptops. Als bedrijf doe jer er dan maar beter goed aan, om mee te gaan met je tijd om zeker te zijn dat je doelgroep niet mist. Je kan kiezen voor native en/of HTML5 app ontwikkeling.

Native

Dit betekent dat we de app ontwikkelen in de codetaal eigen aan het besturingssysteem, bijvoorbeeld Android. Een voordeel is dat je de specifieke API van het besturingssysteem kan aanroepen waarmee je zeer specifiek features kan oproepen. De app kan ook zo gebouwd worden dat deze werkt zonder internet connectiviteit. Nadeel is dat deze duurder zijn in ontwikkeling omdat je de app per besturingssysteem opnieuw dient te ontwikkelen.

HTML5 / JQuery

Dit is de nieuwe standaard die reeds door de meeste browsers (of het nu een Tablet, desktop PC of smartphone is) ondersteunt wordt en zonder rekening te houden met het besturingssysteem (IOS, Android, ...). Het grote voordeel hier is dat je eenvoudiger apps kan ontwikkelen die op alle toestellen kunnen draaien. Een nadeel is dat HTML5 is nog niet helemaal uniform is, dus je bent toch nog steeds gelimiteerd als je app op zoveel mogelijk apparaten moet kunnen werken zonder aanpassingen.
